## <span style="color:#0f766e">1. ✦ 文件定位与总体目标</span>

这份脚本是一个“感知 + 行为”融合的足球踢球流程控制器，目标是：

1. 用 <span style="color:#1d4ed8">YOLO</span>（必要时结合颜色回退）持续检测球和球门。
2. 在接近阶段调用 <span style="color:#0f766e">`approachball.py`</span>（以模块方式内嵌运行，不再单独开进程）。
3. 在接近结束后进入“对齐并射门”状态机，驱动动作组完成踢球。

可以理解为三段式：

1. 颜色锁定与预动作（pre-approach）
2. <span style="color:#0f766e">approachball</span> 接近流程
3. 对齐+踢球（align_and_shoot）

## <span style="color:#0f766e">3. ✦ 启动与路径机制</span>

## <span style="color:#1d4ed8">3.1 ▸ `find_repo_root(start)`（18-23）</span>

逻辑：从当前脚本路径向上查找，遇到包含 `TonyPi` 目录的位置就认为是仓库根目录 `repo_root`。  
如果没找到，则退回到脚本所在目录的父目录。

意义：减少硬编码路径，使脚本可在仓库内不同位置运行。

## <span style="color:#1d4ed8">3.2 ▸ `bootstrap_paths(repo_root)`（26-33）</span>

向 `sys.path` 注入：

1. `repo_root`
2. `repo_root/TonyPi`
3. `repo_root/TonyPi/HiwonderSDK`

意义：保证后续 `hiwonder.*`、`TonyPi.*` 模块可导入。

## <span style="color:#1d4ed8">3.3 ▸ `resolve_default_model(repo_root, model_arg)`（35-46）</span>

模型路径优先级：

1. 命令行 `--model`
2. `models/best.pt`

若都不存在，返回候选1（随后在 `main` 里统一报错）。

## <span style="color:#0f766e">4. ✦ 抽象层设计</span>

## <span style="color:#1d4ed8">4.1 ▸ `Detection` 数据类（49-55）</span>

字段：

1. `name`：类别名
2. `conf`：置信度
3. `cx, cy`：框中心坐标
4. `xyxy`：边界框坐标

这让后续逻辑不依赖 <span style="color:#1d4ed8">YOLO</span> 原始对象，便于统一处理。

## <span style="color:#1d4ed8">4.2 ▸ `CameraAdapter`（58-87）</span>

支持两种源：

1. `source='robot'`：`hiwonder.Camera` 接口
2. 其他：OpenCV `VideoCapture`

外部统一调用：

1. `read()`
2. `close()`

好处：主流程不需要关心底层相机实现差异。

## <span style="color:#1d4ed8">4.3 ▸ `MotionAdapter`（89-127）</span>

核心思想：动作执行可自动降级。

1. 若 `run_on_robot=True`，尝试导入动作组模块：
   - `hiwonder.ActionGroupControl`
   - `TonyPi.HiwonderSDK.hiwonder.ActionGroupControl`
2. 任一失败不会崩溃，回退到 <span style="color:#64748b">dry-run</span>（打印动作而不执行）。

统一接口：

1. `run_action(action, times)`
2. `stand()`（固定执行 `stand_slow`）

## <span style="color:#0f766e">7. ✦ `approachball.py` 的内嵌会话封装</span>

## <span style="color:#1d4ed8">7.1 ▸ `load_approach_module`（242-249）</span>

用 `importlib.util.spec_from_file_location` 动态加载外部脚本为模块。

## <span style="color:#1d4ed8">7.2 ▸ `ApproachBallSession`（202-239）</span>

封装了三个动作：

1. `start(...)`：
   - 设置 `debug`、`align_round`
   - 调用模块的 `init/reset/start`
   - 用锁定颜色设置目标球色
2. `step(frame)`：
   - 把主线程读到的帧交给 <span style="color:#0f766e">`approachball.run(...)`</span>
   - 根据 `kick_ready_to_exit` 判断是否结束
3. `abort()`：异常或退出时调用 `stop()`

要点：不是开新进程，而是“共享主线程相机帧”的 in-process 模式。

## <span style="color:#0f766e">9. ✦ 视觉处理函数</span>

## <span style="color:#1d4ed8">9.1 ▸ `extract_ball_rgb`（302-327）</span>

## <span style="color:#1d4ed8">9.3 ▸ `draw_detection`（353-360）</span>

## <span style="color:#1d4ed8">9.4 ▸ `detect_by_color_fallback`（377-400）</span>

当 <span style="color:#1d4ed8">YOLO</span> 丢球或丢门时可补偿：

1. 红色阈值找球
2. 蓝色阈值找球门

## <span style="color:#0f766e">10. ✦ 预动作逻辑 `run_pre_approach_sequence`（403-501）</span>

这是在正式进入 <span style="color:#0f766e">`approachball`</span> 前的一次快速几何调整。

## <span style="color:#1d4ed8">10.1 ▸ 早退条件</span>

若初始 `ball_cy0 > 0.4*h`，直接跳过预动作。

含义：球已经比较近，没必要做额外横移。

## <span style="color:#1d4ed8">10.2 ▸ 中线附近直冲策略</span>

若当前误差 `|err| <= 0.7 * pre_center_tol`，直接 `go_forward_fast x6`，并触发程序退出标志：

1. `ctx.exit_requested = True`
2. `ctx.phase = done`
3. `ctx.kick_done = True`

这段是一个“快速终止分支”，属于非常激进的策略。

## <span style="color:#1d4ed8">10.3 ▸ 单方向横移 + 过冲一次</span>

若误差明显，选定 `first_side_dir`（只选一次）并持续横移：

1. 直到球越过中线并超过容差（过冲触发）
2. 再补一次同方向动作
3. 结束预动作

最后把结果写入 `ctx.pre_side_move_dir`，供后续搜索方向决策使用。

## <span style="color:#0f766e">11. ✦ 后台状态机 `action_worker`（502-865）</span>

这是接近结束后的“对齐并射门”主决策器，运行在后台线程。

## <span style="color:#1d4ed8">11.1 ▸ 阶段切换前等待</span>

先等待：

1. `approach_started=True`
2. `approach_done=True`
3. 视觉恢复就绪 `vision_ready_after_approach=True`

满足后进入 `phase='align_and_shoot'`。

## <span style="color:#1d4ed8">11.2 ▸ 三层超时保护</span>

1. 若 30s 都没进入 kick 阶段：`go_forward_fast x5` 并结束
2. 程序总运行超过 50s：同样强制结束
3. 动作冷却 `action_cd` 防抖

## <span style="color:#1d4ed8">11.3 ▸ 状态机状态定义</span>

1. `line_intercept`：先对齐“球-门连线与底边交点”
2. `align_goal`：再让球门中心朝向中线
3. `kick_check`：静稳检查 20 帧
4. `kick`：最后距离微调并执行踢球动作

## <span style="color:#1d4ed8">11.4 ▸ 关键几何量</span>

1. `ball_err = ball.cx - center_x`
2. `goal_err = goal.cx - center_x`
3. 估算连线与 `y=h_ref` 的交点 `x_intersect`
4. `inter_err = x_intersect - center_x`
5. `same_side`：球与门是否同侧（相对中心线）

### 交点计算说明

1. 若 `dx` 近 0（连线近垂直），直接令 `x_intersect = ball.cx`
2. 否则按参数方程推到 `y=h_ref` 的交点 x

这比只看 `ball_err` 或 `goal_err` 更接近“射门线路是否穿过中线”的目标。

## <span style="color:#1d4ed8">11.5 ▸ 各状态行为</span>

### <span style="color:#f97316">A) ◦ `line_intercept`</span>

1. 若球和门同侧：原地转向（同侧右转右、同侧左转左）
2. 若异侧但交点误差超阈值：左右平移修正 `inter_err`
3. 满足阈值后转入 `align_goal`

### <span style="color:#f97316">B) ◦ `align_goal`</span>

1. 若前置条件破坏（同侧或交点偏差变大）回退到 `line_intercept`
2. 若 `|goal_err| <= goal_tol`，进入 `kick_check`
3. 否则通过左右转继续把门摆正

### <span style="color:#f97316">C) ◦ `kick_check`</span>

1. 需要连续 20 帧满足交点稳定
2. 任一帧失配则回退到 `align_goal`
3. 达标后进入 `kick`

### <span style="color:#f97316">D) ◦ `kick`</span>

1. 先看球顶点高度 `ball_top_y`，做前后小步距离补偿
2. 距离合适后执行：
   - `stand`
   - `left_shot`
   - `stand`
   - `right_shot`
   - `stand`
3. 写 `ctx.kick_done=True` 并结束线程

## <span style="color:#1d4ed8">11.6 ▸ 丢失目标时的恢复策略</span>

1. 球丢失：
   - `kick_check` 下回退
   - `align_goal` 下若门也丢失则抬头并按预方向搜索；若门在则后退一步
   - 其他情况下根据 `dynamic_round` 或后退动作恢复
2. 门丢失：
   - 抬头 + 根据 `pre_side_move_dir` 方向转向搜索

## <span style="color:#0f766e">12. ✦ 主线程 `main()` 逐步时序（867-1128）</span>

1. 解析参数、定位根目录、注入路径
2. 导入 <span style="color:#1d4ed8">YOLO</span>、加载模型、加载 <span style="color:#0f766e">approachball</span> 模块
3. 初始化相机、动作适配器、共享上下文
4. 预置云台姿态
5. 启动 <span style="color:#f97316">`action_worker`</span> 后台线程
6. 主循环：
   - 读帧
   - <span style="color:#1d4ed8">YOLO</span> 检测 + 颜色回退
   - 更新 `ctx.latest_ball/goal`
   - 可视化覆盖层
   - 当球出现后连续采样颜色，达到 `stable_frames` 锁定颜色
   - 启动 `pre_approach_thread`
   - 预动作结束后启动 `approach_session.start(...)`
   - `phase=approach_running` 时，帧直接喂给 `approach_session.step(...)`
7. 当 `kick_done` 或用户按键退出，进入 `finally`：
   - `stop_event.set()`
   - `worker.join(...)`
   - `approach_session.abort()`
   - 释放相机与窗口

   - 硬件导入失败自动 <span style="color:#64748b">dry-run</span>
   - <span style="color:#1d4ed8">YOLO</span> 丢检时颜色回退
   - 丢球丢门时有恢复策略
3. 参数化程度高：
   - 容差、动作名、动作次数都可命令行调整

## <span style="color:#dc2626">14. ✦ 需要注意的实现细节</span>

1. <span style="color:#f97316">`action_worker`</span> 中 `h_ref` 来自 `getattr(args, 'height', 480)`，但参数里没有 `--height`，因此默认恒为 480。  
   这通常可运行，但如果实际图像高度不是 480，距离阈值会有比例误差。
2. <span style="color:#f97316">`run_pre_approach_sequence`</span> 中“中心附近直接前冲并请求退出”分支非常强，实战中可能跳过后续完整对齐踢球链路。
3. `--line-align-tol` 在当前代码未直接参与决策（主要用的是 `center_align_tol` / `goal_face_tol`），可视为预留或历史参数。
